# Интернет-магазин автозапчастей

Веб-приложение интернет-магазина автозапчастей, разработанное на Django с использованием PostgreSQL в качестве системы управления базами данных.

## Функциональные возможности

### Основные страницы
- **Главная страница** - общая информация о магазине, акции, новинки
- **Каталог товаров** - просмотр всех доступных автозапчастей
- **Корзина** - управление выбранными товарами с расчетом итоговой стоимости
- **Вход/Регистрация** - система аутентификации пользователей
- **История заказов** - просмотр всех предыдущих заказов пользователя (только для авторизованных пользователей)

### Безопасность
- Хэширование паролей пользователей с использованием современных алгоритмов
- Защищенные сессии пользователей
- Безопасное хранение учетных данных
- Защита персональных данных пользователей

### Каталог товаров
Каждая деталь в каталоге содержит:
- **Идентификатор детали** - уникальный артикул товара
- **Изображение детали** - визуальное представление товара
- **Наименование** - название автозапчасти
- **Описание** - подробные характеристики и особенности
- **Наличие** - статус наличия на складе
- **Срок доставки** - время доставки при отсутствии на складе
- **Цена** - стоимость товара

### Поисковая система
Возможность поиска деталей по:
- Наименованию товара
- Идентификатору (артикулу)
- Описанию товара

### Корзина покупок
- Отображение всех добавленных товаров
- Возможность изменения количества
- Удаление товаров из корзины
- Расчет итоговой суммы с учетом:
  - Стоимости всех товаров
  - Стоимости доставки

### Оформление заказа
При оформлении заказа пользователи указывают:
- **Контактные данные**:
  - ФИО получателя
  - Номер телефона
  - Email для уведомлений
- **Адрес доставки**:
  - Город
  - Улица, дом, квартира
  - Почтовый индекс
- **Способ доставки** (выбирается из доступных вариантов)
- **Способ оплаты** (наличные при получении, карта онлайн и т.д.)
- **Комментарий к заказу** (опционально)

### История заказов
Для авторизованных пользователей доступна история заказов, которая включает:
- **Список всех заказов** с возможностью фильтрации по статусу
- **Детальная информация по каждому заказу**:
  - Номер заказа и дата оформления
  - Состав заказа (товары, количество, цена)
  - Статус заказа (обработка, сборка, доставка, выполнен)
  - Информация о доставке (адрес, сроки, трек-номер)
  - Способ оплаты и сумма
- **Возможность повторного заказа** из истории
- **Отслеживание статуса** текущих заказов

## Технологический стек

### Backend
- **Django** - основной фреймворк
- **Django ORM** - работа с базой данных
- **Django Authentication** - система аутентификации
- **Django Sessions** - управление сессиями пользователей

### База данных
- **PostgreSQL** - реляционная СУБД для хранения данных

### Frontend
- **HTML5** - разметка страниц
- **CSS3** - стилизация
- **JavaScript** - интерактивные элементы
- **Bootstrap** (опционально) - для адаптивного дизайна

## Модели данных

### Основные модели:
- **User** - пользователи системы
- **UserProfile** - расширенная информация о пользователе
- **Product** - товары (автозапчасти)
- **Category** - категории товаров
- **Cart** - корзина пользователя
- **CartItem** - элементы корзины
- **Order** - заказы (содержит информацию о пользователе, доставке, оплате)
- **OrderItem** - элементы заказа
- **ShippingAddress** - адреса доставки пользователей
- **OrderHistory** - история изменений статусов заказов

## Административная панель

Панель администратора Django предоставляет возможности:
- Управление товарами (добавление, редактирование, удаление)
- Управление категориями
- Просмотр и управление заказами
- Управление пользователями
- Просмотр истории заказов всех пользователей
- Изменение статусов заказов
- Управление способами доставки и оплаты

## Особенности реализации

### Оформление заказа:
1. Пользователь переходит из корзины на страницу оформления заказа
2. Авторизованные пользователи могут использовать сохраненные данные
3. Форма валидирует все обязательные поля
4. После подтверждения создается заказ и очищается корзина
5. Пользователь получает уведомление на email
6. Заказ появляется в истории заказов

### История заказов:
1. Доступна только авторизованным пользователям
2. Заказы сортируются по дате (новые сверху)
3. Реализована пагинация при большом количестве заказов
4. Возможность фильтрации по статусу заказа
5. Для каждого заказа можно посмотреть детали
