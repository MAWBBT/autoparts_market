## Настройка PostgreSQL

### Шаги настройки

#### 1. Создание базы данных в PgAdmin

1. Откройте PgAdmin
2. Подключитесь к вашему серверу PostgreSQL
3. Создайте новую базу данных:
   - Правой кнопкой на "Databases" → "Create" → "Database..."
   - Имя базы данных: `mystore_db`
   - Owner: `postgres` (или ваш пользователь)
   - Нажмите "Save"

#### 2. Настройка подключения в settings.py

Файл `store/settings.py` уже настроен на PostgreSQL. Проверьте параметры подключения:

```python
DATABASES = {
    'default': {
        'ENGINE': 'django.db.backends.postgresql',
        'NAME': 'mystore_db',      # Имя вашей БД
        'USER': 'postgres',         # Ваш пользователь PostgreSQL
        'PASSWORD': '12345678',     # ⚠️ ИЗМЕНИТЕ НА ВАШ ПАРОЛЬ
        'HOST': 'localhost',
        'PORT': '5432',
    }
}
```

**Важно:** Измените `PASSWORD` на ваш реальный пароль PostgreSQL!

#### 3. Установка зависимостей

Убедитесь, что установлены все необходимые пакеты:

```bash
pip install -r requirements.txt
```

Или установите вручную:
```bash
pip install django djangorestframework djangorestframework-simplejwt psycopg2-binary
```

#### 4. Выполнение миграций

После создания базы данных выполните миграции:

```bash
python manage.py makemigrations
python manage.py migrate
```

#### 5. Создание пользователей

```bash
python manage.py create_users
```

Эта команда создаст:
- Обычного пользователя: `user@example.com` / `12345678`
- Администратора: `admin@example.com` / `admin12345`

#### 6. Запуск сервера

```bash
python manage.py runserver
```

## API для авторизации

### Важно!

API эндпоинты `/api/accounts/login/` и `/api/accounts/register/` принимают **только POST запросы**. 
Если вы откроете их в браузере (GET запрос), получите ошибку 405 "Method Not Allowed" - это нормально!

### Endpoints

#### Регистрация пользователя
**POST** `/api/accounts/register/`

Тело запроса:
```json
{
    "email": "user@example.com",
    "full_name": "Иван Иванов",
    "password": "12345678"
}
```

#### Вход (получение JWT токена)
**POST** `/api/accounts/login/`

Тело запроса:
```json
{
    "email": "user@example.com",
    "password": "12345678"
}
```

Ответ:
```json
{
    "refresh": "eyJ0eXAiOiJKV1QiLCJhbGciOiJIUzI1NiJ9...",
    "access": "eyJ0eXAiOiJKV1QiLCJhbGciOiJIUzI1NiJ9...",
    "email": "user@example.com",
    "full_name": "Иван Иванов",
    "role": "user"
}
```

#### Обновление токена
**POST** `/api/accounts/token/refresh/`

Тело запроса:
```json
{
    "refresh": "..."
}
```

### Использование токена

Для авторизованных запросов добавьте заголовок:
```
Authorization: Bearer <ваш_access_токен>
```

### Примеры использования

#### PowerShell (Invoke-RestMethod)

```powershell
# Регистрация
$body = @{
    email = "user@example.com"
    full_name = "Иван Иванов"
    password = "12345678"
} | ConvertTo-Json

Invoke-RestMethod -Uri "http://127.0.0.1:8000/api/accounts/register/" -Method POST -Body $body -ContentType "application/json"

# Вход
$body = @{
    email = "user@example.com"
    password = "12345678"
} | ConvertTo-Json

$response = Invoke-RestMethod -Uri "http://127.0.0.1:8000/api/accounts/login/" -Method POST -Body $body -ContentType "application/json"
$response | ConvertTo-Json
```

#### JavaScript (fetch)

```javascript
// Вход
fetch('http://127.0.0.1:8000/api/accounts/login/', {
    method: 'POST',
    headers: {
        'Content-Type': 'application/json',
    },
    body: JSON.stringify({
        email: 'user@example.com',
        password: '12345678'
    })
})
.then(response => response.json())
.then(data => {
    console.log('Access token:', data.access);
    console.log('Refresh token:', data.refresh);
});
```

### Валидация

- **Email:** должен содержать символ `@`
- **Пароль:** минимум 8 символов
- Пароли автоматически хэшируются при сохранении

### Ошибки

- **405 Method Not Allowed**: Вы используете неправильный HTTP метод (например, GET вместо POST)
- **400 Bad Request**: Неверные данные (неверный формат email, короткий пароль и т.д.)
- **401 Unauthorized**: Неверные учетные данные при входе

## Админ-панель

### Доступ

Админ-панель доступна только пользователям с ролью **Администратор** (`role='admin'`) или суперпользователям (`is_superuser=True`).

**URL:** `/admin-panel/`

### Функциональность

#### 1. Дашборд (`/admin-panel/`)
- Общая статистика системы
- Количество пользователей, товаров, заказов
- Последние заказы

#### 2. Управление пользователями (`/admin-panel/users/`)
- Просмотр списка всех пользователей
- Поиск по email или имени
- Создание нового пользователя
- Редактирование пользователя:
  - Email, имя
  - **ВАЖНО:** Администратор не может изменять пароль пользователя
  - Роль (Пользователь, Менеджер, Администратор)
  - Статус активности
  - Права доступа (персонал, суперпользователь)
- Удаление пользователя

#### 3. Управление товарами (`/admin-panel/products/`)
- Просмотр списка всех товаров
- Поиск по названию, артикулу, описанию
- Фильтрация по категориям
- Создание нового товара
- Редактирование товара:
  - Категория, артикул, название
  - URL (slug), описание
  - Цена, количество на складе
  - Изображение
  - Срок доставки
  - Доступность
- Удаление товара

#### 4. Управление категориями (`/admin-panel/categories/`)
- Просмотр всех категорий
- Количество товаров в каждой категории
- Создание новой категории
- Редактирование категории (название, URL, описание)
- Удаление категории (с предупреждением о товарах)

#### 5. Управление заказами (`/admin-panel/orders/`)
- Просмотр всех заказов
- Поиск по ID, имени, email, телефону
- Фильтрация по статусу
- Просмотр деталей заказа:
  - Список товаров
  - Данные получателя
  - Изменение статуса заказа
- Удаление заказа

### Статусы заказов

- **В обработке** — новый заказ
- **Принято поставщиком** — заказ принят
- **Доставлен в филиал** — товар доставлен
- **Выдан** — заказ выполнен
- **Отменен** — заказ отменён

### Безопасность

- Доступ только для администраторов (проверка через декоратор `@admin_required`)
- Все формы защищены CSRF-токенами
- Валидация данных на стороне сервера
- Администратор не может изменять пароли пользователей

### Технические детали

- **Декоратор:** `main/decorators.py` — `@admin_required`
- **Views:** `main/views.py` — функции с префиксом `admin_*`
- **Формы:** `main/admin_forms.py`
- **Шаблоны:** `templates/main/admin/*.html`
- **URL:** `main/admin_urls.py` — namespace `admin_panel`

## Описание страниц

### Главная страница

Главная страница содержит информацию о компании, категории товаров и популярные товары. Текст структурирован по разделам: приветствие и краткое описание магазина, история компании и срок работы на рынке, описание предлагаемых товаров и услуг, преимущества сотрудничества.

### Страница "О нас"

Современная страница с информацией о компании:
- Hero-секция с градиентным фоном
- Преимущества компании (6 карточек)
- Статистика (14+ лет, 50K+ клиентов, 10K+ товаров, 100+ городов)
- Миссия и ценности
- Контактная информация

### Страница каталога товаров

Страница каталога содержит табличный список товаров. Каждая строка таблицы представляет один товар и содержит следующие колонки:
- Идентификатор детали (артикул)
- Изображение товара
- Наименование детали
- Текстовое описание
- Информация о наличии на складе
- Срок доставки при отсутствии на складе
- Цена товара

В верхней части страницы расположена поисковая строка с полем ввода и кнопкой "Найти". Поиск осуществляется по трем параметрам:
- Наименованию товара (частичное совпадение)
- Идентификатору товара (точное совпадение)
- Описанию товара (частичное совпадение)

Под описанием каждого товара находятся элементы управления: поле для ввода количества и кнопка "Добавить в корзину".

### Страница корзины

Страница корзины отображает список товаров, добавленных пользователем. Для каждого товара показывается:
- Наименование
- Количество
- Цена за единицу
- Общая стоимость позиции

Рядом с каждым товаром находятся кнопки для удаления и изменения количества.

Под списком товаров отображается расчет стоимости:
- Сумма за все товары
- Стоимость доставки
- Итоговая сумма к оплате

В нижней части страницы находится форма оформления заказа с полями для указания способа оплаты, адреса доставки и персональных данных пользователя. Для авторизованных пользователей часть данных подставляется автоматически из профиля.

### Страница входа в систему

Страница входа содержит форму с двумя полями ввода:
- Логин (email)
- Пароль

Под формой находятся ссылки:
- "Зарегистрироваться" для перехода на страницу регистрации
- "Забыли пароль" для восстановления доступа

### Страница регистрации

Страница регистрации содержит форму с полями для ввода:
- ФИО
- Email
- Телефон (необязательно)
- Пароль
- Подтверждение пароля

Внизу формы расположен чекбокс для согласия с пользовательским соглашением и политикой конфиденциальности, а также кнопка "Зарегистрироваться".

### Страница профиля

Страница профиля доступна только авторизованным пользователям и содержит две формы:

1. **Редактирование личных данных:**
   - Email
   - Полное имя
   - Телефон

2. **Изменение пароля:**
   - Текущий пароль
   - Новый пароль
   - Подтверждение нового пароля

Также на странице есть ссылки на:
- Историю заказов
- Админ-панель (для администраторов)
- Выход из системы

### Страница истории заказов

Страница истории заказов доступна только авторизованным пользователям. Для каждого заказа отображается:
- Номер заказа
- Дата оформления
- Текущий статус
- Состав заказа (список товаров с количеством и ценой)
- Общая сумма

Статусы заказов:
- "В обработке"
- "Принято поставщиком"
- "Доставлен в филиал"
- "Выдан"
- "Отменен"

Для каждого заказа доступна кнопка "Подробнее" для просмотра деталей.

### Страница восстановления пароля

Процесс восстановления пароля состоит из нескольких шагов:

1. **Запрос сброса** (`/password_reset/`):
   - Пользователь вводит свой email
   - Если email существует, отправляется письмо со ссылкой

2. **Письмо на email**:
   - Содержит ссылку для сброса пароля
   - Ссылка действительна ограниченное время

3. **Установка нового пароля** (`/reset/<uidb64>/<token>/`):
   - Пользователь вводит новый пароль дважды
   - Пароль должен быть минимум 8 символов

4. **Подтверждение** (`/reset/done/`):
   - Сообщение об успешной смене пароля
   - Ссылка для входа в систему

## Модели данных

### Основные модели:

- **User** (accounts.User) — кастомная модель пользователя
  - email (уникальный)
  - full_name
  - role (user, manager, admin)
  - is_active
  - is_staff
  - is_superuser

- **UserProfile** (main.UserProfile) — расширенная информация о пользователе
  - user (OneToOne)
  - full_name
  - phone

- **Category** (products.Category) — категории товаров
  - name
  - slug (уникальный)
  - description

- **Product** (products.Product) — товары (автозапчасти)
  - category (ForeignKey)
  - article
  - name
  - slug (уникальный)
  - description
  - price
  - image
  - stock
  - delivery_days
  - available
  - created, updated

- **Cart** (cart.Cart) — корзина пользователя
  - user (ForeignKey, опционально)
  - session_key (для гостей)
  - created, updated

- **CartItem** (cart.CartItem) — элементы корзины
  - cart (ForeignKey)
  - product (ForeignKey)
  - quantity

- **Order** (cart.Order) — заказы
  - user (ForeignKey, опционально)
  - session_key (для гостей)
  - status
  - full_name, phone, email
  - city, address, postal_code
  - payment_method, delivery_method
  - comment
  - total_products, delivery_cost, total
  - created

- **OrderItem** (cart.OrderItem) — элементы заказа
  - order (ForeignKey)
  - product (ForeignKey)
  - quantity
  - price

## Функциональные особенности

### Система аутентификации

- Кастомная модель пользователя с email в качестве username
- Валидация email (символ @) и пароля (минимум 8 символов)
- Хэширование паролей (Django PBKDF2)
- JWT токены для API
- Восстановление пароля через email
- Изменение пароля в профиле

### Корзина покупок

- Корзина привязана к учетной записи пользователя (для авторизованных)
- Для гостей корзина сохраняется в сессии
- Товары остаются в корзине после выхода из системы
- Возможность изменения количества и удаления товаров
- Расчет итоговой суммы с учетом доставки

### Поисковая система

- Поиск по трем критериям одновременно:
  - Названия товаров (частичное совпадение)
  - Идентификаторы (точное совпадение)
  - Описания товаров (частичное совпадение)
- Фильтрация по категориям
- Результаты отображаются в табличной форме

### Оформление заказа

1. Пользователь переходит из корзины на страницу оформления заказа
2. Авторизованные пользователи могут использовать сохраненные данные
3. Форма валидирует все обязательные поля
4. После подтверждения создается заказ и очищается корзина
5. Пользователь получает уведомление на email (если настроено)
6. Заказ появляется в истории заказов с статусом "В обработке"

### История заказов

- Доступна только авторизованным пользователям
- Заказы сортируются по дате (новые сверху)
- Реализована пагинация при большом количестве заказов
- Возможность фильтрации по статусу заказа
- Для каждого заказа можно посмотреть детали

### Безопасность

- Хэширование паролей пользователей
- Защищенные сессии пользователей
- Безопасное хранение учетных данных
- Защита персональных данных пользователей
- CSRF защита всех форм
- Валидация данных на стороне сервера
- Администратор не может изменять пароли пользователей
